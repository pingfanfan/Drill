# Usage Guide

## Basic Usage

### Starting a Drill Session

Simply use the trigger phrase:

```
"Drill into [topic]"
"Drill down on [topic]"
"Let's drill [topic]"
"Drill my [idea/strategy/plan]"
```

### Examples

**Market Analysis**:
```
"Drill into the AI-powered customer support market"
```

**Competitive Strategy**:
```
"Drill down on how we should position against Notion"
```

**Investment Prep**:
```
"Drill my pitch deck before my investor meeting"
```

**Business Model**:
```
"Let's drill this business model"
```

## Advanced Usage

### With Context

Provide background for better analysis:

```
"Drill into the fintech space. We're a 10-person team building
for mid-market CFOs with limited technical resources."
```

### With Constraints

Specify what you're trying to decide:

```
"Drill this market opportunity. I'm specifically trying to decide
whether to pivot from B2C to B2B."
```

### With Materials

If you already have research:

```
"Drill into this. Here are my materials: [paste URLs or text]"
```

## Understanding Output

### Materials Collected

Drill lists what it gathered:
```
## 🔍 Materials Collected
- 5 competitor websites
- 23 user reviews
- 4 community discussions
```

### Raw Observations

Specific evidence points with citations:
```
## 📋 Raw Observations
1. Competitor X emphasizes Y but buries pricing
2. User reviews mention Z 7 times
```

### Unspoken Consensus

What insiders know:
```
## 🎯 Unspoken Consensus
The real job is not [surface need] but [deeper need].
```

### Structural Assumptions

The 3 load-bearing assumptions:
```
## 🏗️ Structural Assumptions
Assumption 1: [X]
- If false when: [condition]
- Fragility: [assessment]
```

### Adversarial Challenges

Questions that would destroy your idea:
```
## ⚔️ Adversarial Challenges
Q1: "[Killer question]"
- Evidence: [from materials]
- Implication: [if unaddressed]
```

### Strategic Implications

What to do:
```
## 💡 Strategic Implications
What survived: [validated points]
What emerged: [new insights]
Next questions: [to explore]
```

## Tips for Best Results

### 1. Be Specific

**Better**: "Drill into the market for AI-powered code review tools for enterprise teams"
**Less effective**: "Drill into AI"

### 2. Provide Context

**Helpful**: "We're a Series A startup with 6 months runway"
**Context helps**: Drill calibrates adversarial intensity

### 3. Follow Up

After initial analysis:
- "Drill deeper on Assumption 2"
- "What would make Challenge 4 the most dangerous?"
- "Expand on the user sentiment analysis"

### 4. Iterate

Use Drill outputs as inputs:
```
"You said [insight]—drill into that specifically"
```

## Fallback Modes

### When Automated Collection Fails

Drill automatically switches to user-assisted mode:

```
AI: "I'm experiencing connectivity limitations. Let me work with
     you to gather materials."

You: [provide URLs or text]

AI: [proceeds with analysis]
```

### Working with Limited Materials

Even with 1-2 sources, Drill provides value:
- Structures your thinking
- Identifies gaps to fill
- Applies adversarial lens to your knowledge

### Request Specific Depth

```

"Deep drill: full 5-layer analysis"
```

### Request Specific Focus

```
"Drill focusing on competitive threats"
"Drill emphasizing regulatory risks"
```

### Request Format

```
"Drill and give me bullet points only"
"Drill with detailed citations"
```

## Troubleshooting Usage

### Analysis feels shallow

**Cause**: Limited materials collected
**Fix**: Provide specific sources or ask for deeper drill

### Not adversarial enough

**Cause**: Confirmation bias in your framing
**Fix**: Say "be more aggressive" or "challenge me harder"

### Too long

**Cause**: Broad topic
**Fix**: Narrow scope: "Drill just the pricing strategy"

### Missing domain knowledge

**Cause**: Drill has general patterns, not specific expertise
**Fix**: Provide domain-specific materials or add domain guide
